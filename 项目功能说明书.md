# PromptHub 项目功能与架构说明书 (v0.5.7)

本项目是一个**本地优先（Local-first）**的提示词（Prompt）与 AI 智能体/技能（AI-Skill & MCP-Agent）生命周期管理平台。支持跨平台桌面端（Electron）、自托管服务端（Hono Web）以及独立的终端命令行工具（CLI）。

---

## 1. 整体系统架构与目录结构

本项目采用多包单仓（Monorepo）的组织形式，使用 `pnpm workspace` 维护，核心代码依功能逻辑划分为应用层与共享包层。

```text
PromptHub/
├── apps/
│   ├── desktop/            # Electron 桌面端 GUI 应用
│   │   ├── src/main/       # 主进程：数据库连接、文件系统监控、原生 IPC 处理器、Agent 网关
│   │   ├── src/renderer/   # 渲染进程：React 页面 (Zustand 状态管理, Tailwind CSS)
│   │   └── src/preload/    # 安全桥接层：暴露 window.api 原生调用
│   ├── web/                # 自托管网页端应用 (Hono + React Web Client)，支持 Docker 独立部署
│   └── cli/                # 独立的终端命令行管理工具 (prompthub CLI)
└── packages/
    ├── db/                 # SQLite 存储层：跨平台 WASM 适配器、自动迁移、数据表操作类 (DB Layer)
    ├── core/               # 核心业务逻辑：路径解析、安全扫描、IDE 规则同步、Agent 本地网关管理
    └── shared/             # 共享定义：跨平台常量、共享数据类型 (TS)、IPC 通道名称
```

### 依赖流动方向
* `apps/*` $\rightarrow$ 引入 `packages/*`
* `packages/core` $\rightarrow$ 引入 `packages/db` 与 `packages/shared`
* `packages/db` $\rightarrow$ 引入 `packages/shared`
* 共享包（`packages/*`）绝不能引入任何特定应用层（`apps/*`）的代码，保证底层数据存储与核心业务引擎的纯粹性与可重用性。

## 2. 数据库设计与核心存储机制

数据库统一使用 **SQLite**，物理文件名为 `prompthub.db`。在实现上极具特色：

### 2.1 WASM 无编译跨平台驱动
为了解决经典 SQLite 驱动（如 `better-sqlite3`）在不同平台（尤其是 Windows ARM64 或 Apple Silicon）因原生 C++ 模块编译失败而导致的 DLL 缺失错误，项目在 `packages/db/src/adapter.ts` 中使用 **`node-sqlite3-wasm` (纯 WebAssembly 引擎)**。
项目自研了 `DatabaseAdapter` 桥接层，对外提供完全兼容 `better-sqlite3` 的 API，在 Electron 主进程与 Node.js 网页服务端都做到了**“零编译，开箱即用”**。

### 2.2 全文检索与触发器设计 (FTS5)
系统利用 SQLite FTS5 虚拟表 `prompts_fts` 提供快速的本地提示词搜索：
* **关联映射**：将 `prompts` 表的 `title`, `description`, `system_prompt`, `user_prompt`, `tags` 列映射进全文索引。
* **同步机制**：在主表上配置了三个自动触发器：`prompts_ai` (AFTER INSERT), `prompts_ad` (AFTER DELETE), `prompts_au` (AFTER UPDATE)。每当提示词被编辑，全文索引将无感地保持原子级实时同步。

### 2.3 原子性安全迁移与备份
在每次初始化数据库时，[init.ts](file:///d:/Pyprojects/PromptHub-main2/packages/db/src/init.ts) 会读取 `PRAGMA table_info` 探针检查表结构。若发现包含落后版本的数据结构：
1. **安全物理拷贝**：在同目录下拷贝一份带有时间戳的物理备份，形如 `prompthub.db.backup-2026-06-24-15-57-40`。
2. **事务迁移**：所有的 `ALTER TABLE` 语句被包裹在一个事务中执行（`db.transaction()`），如果中途发生异常则自动回滚，杜绝因中断产生的坏库。

### 2.4 数据表结构一览 (DDL Schema)
* **`prompts` 与 `prompt_versions`**：存储提示词的核心属性（收藏状态、置顶状态、树形多级嵌套 `parent_id`、多语言对比字段 `system_prompt_en`/`user_prompt_en`）及修改产生的历次版本历史快照。
* **`folders`**：树状分类文件夹，支持级联删除 (`ON DELETE CASCADE`) 和前端拖拽排序属性 `sort_order`。
* **`skills` 与 `skill_versions`**：用于 AI Agent 技能定义，包含本地仓库路径 `local_repo_path`、安装哈希指纹 `directory_fingerprint`、前置条件与兼容描述，以及 AI 安全扫描评级 `safety_level`。
* **`rules` 与 `rule_versions`**：追踪各大 IDE AI Agent 配置（如 `.cursorrules`，`CLAUDE.md`）的同步状态 (`sync_status`) 及其历次重写历史。
* **`users`, `refresh_tokens`, `user_settings`, `settings`**：支持网页端自托管服务的多租户认证，支持带有 `last_active_at` 滑动过期的 JWT 校验会话体系。

## 3. 核心功能模块

### 3.1 提示词生命周期与版本归档 (Prompts & Folders)
* **嵌套结构**：支持目录级嵌套（Folders 树）与提示词之间的主子嵌套关系（Prompts 树）。
* **多版本回滚**：提示词表与历史版本表存在一对多级联关联。每次修改核心提示词文本，主进程都会自动创建一个新版本并自增 `current_version`，用户可随时一键回滚（Rollback）至任意历史快照。
* **富媒体与变量**：提示词可绑定图片、视频媒体资产，并支持声明局部插值变量（`variables`），用于大模型上下文拼装。

### 3.2 AI 技能库与 MCP 服务生态 (Skills & MCP)
* **内置/外部商店导入**：支持从官方注册表或第三方商店（如 **ClawHub**）一键拉取技能。拉取支持增量比对 `installed_content_hash` 以触发更新通知。
* **Model Context Protocol (MCP) 集成**：支持直接关联本地文件夹运行。客户端会基于绑定的 Python 虚拟环境（`agent-venv`）拉取执行依赖，并由本地网关（Agent Gateway）托管启动，从而将普通指令包暴露给 IDE（如 Cursor）作为真实的本地工具链使用。
* **AI 驱动的安全沙箱扫描**：导入或安装任何技能包时，由大模型/合规工具对包含的代码和指令进行前置安全审计（Safety Scan），评估出 `safety_level`（安全、警告、高风险、拦截）。若检测到包含高危险级执行指令（例如未经确认的 `rm`、`exec`）或网络穿透行为，将自动拦截运行。

### 3.3 开发规程与 IDE 规则文件同步 (Rules & Cursorrules)
支持管理主流 IDE 与 AI Agent 的全局/项目级提示词上下文规则：
* **支持平台清单**：包括 **Claude Code** (`CLAUDE.md`), **Windsurf** (`global_rules.md`), **Gemini CLI** (`GEMINI.md`), **Codex CLI/OpenCode** (`AGENTS.md`) 等。
* **健康与防篡改监控**：项目会定期对托管规则副本与磁盘上真实的 IDE 配置文件（如 `~/.claude/CLAUDE.md`）进行 SHA-256 哈希比对。自动划分四种同步状态：
  * `synced`：两端一致。
  * `target-missing`：磁盘目标文件被删。
  * `out-of-sync`：磁盘目标文件已被手动修改，产生冲突。
  * `sync-error`：因读写权限引起的同步错误。
* **冲突化解方案**：用户编辑后，系统提供“覆盖到磁盘”或“从磁盘恢复”两种覆盖和解决冲突的策略。

### 3.4 本地 Agent 网关 (Agent Gateway)
* 桌面端主进程会拉取内置的独立虚拟 Python 环境 `agent-venv`。
* 独立子进程启动 `run_gateway.py`（基于 FastAPI + Uvicorn），为每个启用的技能项目分配不同的端口（从 `18792` 开始递增）。
* 网关提供了统一的本地 HTTP 端点 `/api/health` 以及消息通道，使外部大模型和 IDE 工具能够直接与本地执行引擎通信。

### 3.5 整包备份、恢复与多租户同步 (Backup & Sync)
* **加密导出**：本地数据可被导出为整包格式（ZIP / GZ），支持导出媒体资产、AI 配置和个性化设置。支持用户设置自定义密码，使用 **AES-GCM (256-bit)** 对核心 JSON 文本进行强加密，附带 SHA-256 数据指纹校检。
* **云端增量同步 (PUT /data)**：客户端与服务端可通过 REST 接口进行同步。同步采用基于 `updatedAt` 时间戳的合并策略，只有客户端比对后最新修改的数据行才会覆盖服务端；同时，对历史版本数据（Versions）采用 `DELETE` 冲突版本后 `INSERT OR IGNORE` 的方式重新归整，保证合并的高一致性与幂等性。

## 4. 开发与测试质量红线 (Engineering Rules)

项目为保障多包协作和本地优先存储的稳定性，在 [AGENTS.md](file:///d:/Pyprojects/PromptHub-main2/AGENTS.md) 中规定了强制的工程红线：

1. **Mandatory Change Gate (开发前关卡)**：
   若涉及到数据库表修改、API 契约变更、跨包行为（`packages/*`）以及多文件重构，**必须在开发前**在 `spec/changes/active/<change-key>/` 下创建对应方案说明，并以**中文**撰写 `proposal.md`、`spec.md` 及设计任务，获得评审通过后方可执行。
2. **测试先行与覆盖率准入**：
   * 所有的 Bug 修复必须先编写能够复现该 Bug 的失败测试（vitest / playwright）。
   * **新增或修改的生产代码必须达到 100% 的行、函数、分支和条件覆盖率**。
   * 核心边界模块（数据库操作类、文件持久化、同步协议、安全扫描等）合并前强制要求其新增行为达到 100% 分支覆盖率。
