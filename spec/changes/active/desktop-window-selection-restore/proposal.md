# Proposal

## Why

当前桌面端在窗口最小化后重新激活时，提示词列表经常丢失上一次的激活项，上下文体验会退回到列表顶部/第一条附近。这会打断用户连续浏览同一文件夹内多个提示词的流程。

## Scope

- In scope:
- 识别窗口重新显示、重新聚焦、后台恢复后提示词选中态丢失的根因
- 修复当前文件夹内上一次选中 prompt 的恢复逻辑
- 补充相关回归测试
- Out of scope:
- 不处理跨文件夹恢复到历史选中项
- 不实现完整滚动位置恢复或 DOM 级键盘焦点恢复

## Risks

- 如果恢复逻辑没有校验当前筛选/文件夹上下文，可能把选中态恢复到用户当前看不到的 prompt
- 如果把瞬时 UI 选中态全部持久化，可能引入旧状态污染

## Rollback Thinking

- 回退到仅在当前会话内维护选中态，不做恢复补偿
