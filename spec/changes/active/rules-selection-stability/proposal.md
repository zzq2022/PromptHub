# Proposal

## Why

Rules 视图在保存或异步加载后存在选中态漂移风险。用户刚编辑完某个平台规则后，界面可能被晚到的异步读取结果或默认选中逻辑冲掉，表现为当前规则突然切换到别的平台条目。

## Scope

- 修复 rules store 的异步选中/加载竞态。
- 保存当前规则后明确保留当前选中项。
- 为“保存后保持选中项”和“陈旧读取结果不覆盖新选择”补回归测试。

## Risks

- 读取防抖/防陈旧保护如果处理不当，可能让真实刷新结果被错误忽略。
- force reload 路径需要保留手动 rescan 的刷新能力。

## Rollback

- 回滚 rules store 的请求序号保护与相关回归测试。
