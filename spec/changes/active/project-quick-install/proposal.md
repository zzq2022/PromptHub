# Proposal

## Why

在 PromptHub 的日常使用中，“我的 Skill”是当前用户管理所有私有 Skill 的中心汇集地。目前的“安装到平台”弹窗仅支持安装到全局平台（如 Claude Code、Cursor 等），这给需要将私有 Skill 分发到多个本地开发项目的用户带来了不便（需要手动切换到项目分发，或每次手动操作）。

为了优化体验，我们希望将本地已登记的项目也作为安装目标，整合到同一个“安装到平台”快速安装弹窗内，并默认以“软链接（Symlink）”形式将 Skill 链接到项目的 `.agents/skills` 或配置的部署目标中，确保单源同步和实时响应。

## Goals

- 在“安装到平台”快速安装弹窗中展现已注册项目列表。
- 支持在同一个弹窗内，同时勾选 Agent 平台和本地开发项目进行批量部署。
- 项目的安装方式默认且强指定为“软链接（Symlink）”，以支撑“我的 Skill 是所有私有 Skill 汇集地”的心智。
- 保证当项目内已安装了该 Skill 时，置灰显示“已安装”，不可重复勾选。

## Non-Goals

- 本轮不修改平台（Claude Code 等）的默认安装首选项配置，仅针对项目的安装强制要求软链接。
- 本轮不做项目目标的自定义过滤和添加，只选用项目本身预设的 `deployTargets` 进行多选或单选安装。
