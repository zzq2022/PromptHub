# 网页端页面整合与技能中心重构规格说明书

## 1. 行为约束与产品边界

1. **公开与保护边界**：
   - 访客在未登录时，必须能成功渲染 `/`（技能中心首页）并正常运行查询、搜索、查看详情及下载 ZIP。任何需要身份认证的操作（如导入至工作区、进入 Dashboard、进入同步工作区）必须跳转至 `/login` 拦截。
2. **私有资产保护**：
   - 非所有者调用私有技能接口应返回 404，防止信息泄露。
   - 匿名访客在首页点击下载时，只能下载 `visibility = 'shared'` 并且 `approval_status = 'approved'` 的技能。
3. **数据原子性**：
   - 用户提审状态修改、新技能在线创建的数据库写入，必须使用现有的 `packages/db` 中提供的 SQLite 适配器完成。

---

## 2. 验收标准 (Acceptance Criteria)

### 场景 1：访客浏览与搜索公共技能
- **GIVEN** 用户未登录，访问 `http://localhost:5174/`
- **WHEN** 页面加载
- **THEN** 渲染公开技能中心。顶部导航栏右上角显示“登录”按钮，不显示“个人主页”、“提示词工作区”。
- **WHEN** 在搜索框输入 "MCP" 并回车
- **THEN** 页面只展示名称或描述匹配 "MCP" 且状态为已公开（Approved）的技能卡片。

### 场景 2：访客查看详情与匿名下载
- **GIVEN** 用户未登录，浏览公共技能卡片
- **WHEN** 点击卡片“查看详情”
- **THEN** 弹出 Modal，正确渲染 `SKILL.md` Markdown 内容。
- **WHEN** 点击“下载”
- **THEN** 浏览器发起 ZIP 下载请求成功，无需登录令牌。

### 场景 3：已登录用户管理自己技能
- **GIVEN** 用户已登录，访问 `/dashboard`
- **WHEN** 点击“新建技能”并填写表单保存
- **THEN** 发送 `POST /api/skills` 并在“我的技能”列表刷新显示该条目。
- **WHEN** 对私有技能点击“提交审核”
- **THEN** 发送 `/api/skillhub/:id/publish`，技能状态变更为 `待审核 (Pending)`，且该技能不会立即出现在 `/` 首页（直到管理员批准）。

### 场景 4：工作区与主导航互通
- **GIVEN** 用户已登录，点击顶栏的“提示词工作区”
- **WHEN** 路由跳转到 `/workspace`
- **THEN** 加载并渲染完整的原桌面端工作区，顶部导航正常提供返回技能中心和个人主页的链接。
