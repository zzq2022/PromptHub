# Proposal

## Why

Prompt 桌面工作区当前有三个交互问题叠在一起：

- AI 测试抽屉会在文本 Prompt 上也显示“测试生图”，让模式边界变得混乱。
- 聊天 / 多模型测试原本支持临时图片附件，但当前 UI 只在 image Prompt 上渲染参考图入口，导致多模态文本测试无法插图。
- 新建 / 编辑 Prompt 弹窗把描述、类型、媒体、文件夹、标签、来源、备注等全部挤在同一个属性折叠区里，核心创作流和扩展元信息混在一起，用户难以快速判断“先写什么、去哪加参考图”。

## Scope

- In scope:
  - 修正 AI 测试抽屉的模式可见性，只让 image Prompt 暴露生图测试。
  - 恢复并明确文本 Prompt 的图片附件入口，用于多模态聊天测试和多模型对比。
  - 优化 Prompt 弹窗结构：新建时突出标题、Prompt 类型与用户提示词；编辑时保留描述、Prompt 类型与 image Prompt 参考图的基础编辑上下文。
  - 调整相关用户文案，降低“媒体 / 生图 / 参考图”认知歧义。
  - 添加聚焦回归测试并记录验证结果。
- Out of scope:
  - 新增视频 / 音频多模态测试协议。
  - 改动 Prompt 存储结构或数据库 schema。
  - 重写整个 Prompt 编辑器或引入新的富文本编辑能力。

## Risks

- Create/Edit 两个弹窗结构高度重复，但本轮又允许首屏层级按场景分化；如果测试和 spec 没跟上，后续很容易把有意差异误回滚。
- AI 测试抽屉需要同时兼顾 image Prompt、生图模型和文本 Prompt 的多模态聊天附件，条件判断容易再次互相污染。
- 文案调整会影响多个 locale，需要保持 key 一致，避免引入 i18n 回归。

## Rollback Thinking

若需要回滚，可撤销：

- AI 测试抽屉的模式显示条件修改。
- 新建 / 编辑 Prompt 弹窗的布局重排。
- 新增文案 key 与测试。

本次不涉及数据迁移，回滚不会影响已有 Prompt 数据。
