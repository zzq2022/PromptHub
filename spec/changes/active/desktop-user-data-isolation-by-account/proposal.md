# 提案：桌面端多账号本地数据物理隔离及同步权限联动

## 1. 目标与背景
目前，PromptHub 桌面端在进行远程同步和备份时存在所有权冲突和数据混杂的问题。本提案旨在通过根据登录的自部署/云端账户验证状态动态划分本地数据根路径，实现：
1. **数据完全隔离**：不同账号的本地数据（数据库、技能库、提示词等）物理路径隔离。
2. **同步权限与验证状态联动**：
   - **未验证/本地离线状态**：如果用户未在“数据同步-自部署”中配置或通过测试，应用以**当前操作系统的登录用户名/用户ID**（如 `Administrator`）隔离和存储数据。但在 UI 界面和 IPC 中**禁用并阻止任何与服务端的同步、备份、推送、拉取等远程操作**（仍允许本地文件的导入导出）。
   - **验证成功状态**：一旦用户在同步配置中配置并点击“测试连接”验证通过后，自动切换至专属账号 ID（如 `md5(username)`）的物理目录下，并**解锁全部远程同步、推送、拉取和备份功能**。

## 2. 核心原理
1. **未激活同步状态（访客态）**：
   - 本地根目录：`C:\Users\<OS_Username>\AppData\Roaming\PromptHub\users\<OS_Username>`（通过 `os.userInfo().username` 获取）。
   - 界面上禁用远程备份、推送等操作。
2. **同步测试验证成功（激活态）**：
   - 当 WebDAV “测试连接”返回成功时，前端触发 IPC `database:switch-account`，参数为新账号的 ID（由用户名/凭证生成）。
   - 主进程重载数据库和路径，将根目录切换至：`C:\Users\<OS_Username>\AppData\Roaming\PromptHub\users\<accountId>`。
   - 界面解锁远程备份和数据推送/拉取操作。
3. **断开/重置同步**：
   - 当用户注销或清除同步配置时，切换回系统登录用户名数据路径，再次禁用远程同步。
