# Proposal

## Why

当前桌面端对用户数据目录的语义并不统一：

- 主数据库位于 `userData/prompthub.db`
- 工作区与资源文件位于 `userData/data/*`

这导致用户很容易把 `data/` 误认为“完整数据目录”。当用户跨版本迁移时，如果只复制 `data/`，新版本启动后会缺少真正的主数据库，只能依赖工作区文件重建数据，进而表现为“部分 prompt 丢失”。

## Scope

- 统一桌面端用户数据布局语义，使“数据目录”包含主数据库与工作区数据
- 设计从旧布局到新布局的安全迁移方案
- 保持现有恢复、备份、数据路径切换能力可用
- 补充与数据布局相关的回归验证计划

## Risks

- 会影响桌面端数据库初始化路径、恢复路径、升级备份与数据路径切换逻辑
- 如果迁移顺序处理不当，可能导致升级时读到旧路径与新路径的混合状态
- 需要兼容已经存在的旧版 `userData/prompthub.db` 与 `userData/data/*`

## Rollback

- 保留旧路径探测与回退能力，在新布局迁移失败时继续从旧布局启动
- 在完成自动迁移前创建升级快照，允许用户恢复到迁移前状态
