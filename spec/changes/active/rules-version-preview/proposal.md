# Proposal

## Why

Rules 工作台已经展示版本快照，但当前 UI 只渲染静态列表，用户无法点击查看历史内容，也无法把历史版本恢复到当前草稿。这个落差会让用户误以为版本历史已经完整可用，实际却无法完成核心回看流程。

## Scope

- In scope:
- 为 Rules 工作台补齐版本快照的可点击预览能力
- 为 Rules 工作台补齐“恢复到当前草稿”的显式操作
- 调整右侧编辑区支持“当前草稿 / 历史预览”双模式
- 补齐相关文案与回归测试
- Out of scope:
- 新增单独的规则历史 IPC 接口
- 直接从历史快照一键覆盖磁盘文件
- 规则快照 diff 对比视图

## Risks

- 历史预览如果直接复用当前编辑框，容易误把只读预览做成可编辑状态
- 恢复操作如果直接写盘，会意外覆盖用户当前尚未确认的草稿

## Rollback Thinking

- 若交互效果不理想，可回退到只显示静态版本列表，但必须同时移除误导性的历史查看文案
