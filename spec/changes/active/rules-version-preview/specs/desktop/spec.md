# Delta Spec

## Added

- Rules 工作台中的版本快照在 UI 上必须可点击，并能展示所选快照的具体内容。
- 当用户查看某个历史快照时，右侧规则内容区必须切换到只读预览模式，避免误把历史内容当作当前草稿直接修改。
- 用户必须可以将某个历史快照恢复到当前草稿，但恢复操作不应自动写入磁盘文件。

## Modified

- 版本快照区不再只是被动展示历史条目，而是承担“浏览历史 -> 恢复到草稿”的交互职责。

## Removed

- 无。

## Scenarios

- 当用户点击版本快照列表中的某一项时：
- 系统展示该快照内容
- 右侧编辑区进入只读预览模式
- 用户可以返回当前草稿或将该快照恢复到草稿

- 当用户点击“恢复到草稿”时：
- 当前编辑草稿内容更新为所选快照内容
- 系统不会自动保存到磁盘
- 后续仍需用户显式点击保存
