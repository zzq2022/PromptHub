# Design

## Root Cause

- 更新弹窗依赖通用 `Modal`，其内容区作为纵向 flex 子项缺少 `min-h-0`，在发布说明和备份提醒同时出现时，内容无法正确收缩到可滚动区域内。
- 备份提醒区同时展示标题、说明、缺失状态、确认提示 4 层信息，且其中两层重复表达“安装前仍会自动创建本地快照”。
- 勾选确认使用原生 `input[type="checkbox"]`，没有复用桌面端已有 `ui/Checkbox` 视觉样式。

## Approach

- 在 `Modal` 内容区补充 `min-h-0`，让长内容弹窗进入内部滚动而非整体溢出。
- 让更新弹窗头部动作区支持换行，并在窄宽度下将“检查更新”按钮收敛为图标 + 可访问标签，降低头部横向占用。
- 重组备份提醒区：保留标题，使用一条状态消息承载“缺少备份 / 已备份”的核心信息，移除额外确认警告文本。
- 使用 `ui/Checkbox` 替换原生复选框，保持更新弹窗与桌面端表单控件的一致性。

## Tradeoffs

- 采用通用 `Modal` 修复可以一次解决同类长内容弹窗问题，但会轻微改变其他弹窗的纵向收缩行为。
