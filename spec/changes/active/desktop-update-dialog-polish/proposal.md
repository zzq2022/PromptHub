# Proposal

## Why

修复桌面端更新弹窗在高内容场景下的布局溢出，并收敛升级备份提醒的信息密度，让升级确认区域更易读且与现有 UI 组件风格一致。

## Scope

- In scope:
- 调整更新弹窗的头部与内容区布局收缩行为，避免内容较长时溢出。
- 精简升级备份提醒区的层级和冗余说明。
- 将更新弹窗中的原生复选框替换为项目内统一复选框样式。
- Out of scope:
- 更改更新流程、备份逻辑或安装行为。
- 改写发布说明内容来源。

## Risks

- 通用 `Modal` 的内容区收缩行为变化可能影响其他长内容弹窗的滚动表现。

## Rollback Thinking

- 若发现通用 `Modal` 收缩调整影响其他弹窗，可仅保留 `UpdateDialog` 内的局部样式修复并回退通用布局改动。
