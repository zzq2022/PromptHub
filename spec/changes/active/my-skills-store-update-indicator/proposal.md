# Proposal

## Why

用户已经可以从 Skill Store 安装技能，也可以在商店详情里手动检查更新，但如果平时停留在“我的 Skills”视图，无法一眼看出哪些商店安装的技能已经有新版本。这导致用户必须逐个回到商店详情手动确认，成本高且容易漏掉更新。

## Scope

- 仅对“从商店安装”的 Skill（有 `registry_slug`）做更新提示。
- 在“我的 Skills”列表视图和画廊视图中，为存在可用商店更新的 Skill 显示轻量呼吸灯提示。
- 复用现有商店源同步能力，让“我的 Skills”也能拿到远端商店目录数据，而不是依赖用户先打开商店。
- 不在本次范围内实现“一键全部更新”或复杂的全局通知中心。

## Risks

- 若直接在每个 Skill 上单独拉远端内容，可能造成过多网络请求；需要优先基于已缓存的远端商店目录做版本级提示。
- 需要避免对本地导入/手工创建 Skill 误报更新。

## Rollback

- 移除“我的 Skills”里的更新提示 UI。
- 撤回新增的远端商店同步复用逻辑，恢复为仅商店页内部维护远端源缓存。

## Impacted User Flows

- 用户打开“我的 Skills”时，可以直接看到哪些商店安装的 Skill 有新版本。
- 用户切换列表/画廊视图时，更新提示保持一致。
- 用户未打开商店页时，仍可收到基于商店目录缓存/同步结果的更新提示。
