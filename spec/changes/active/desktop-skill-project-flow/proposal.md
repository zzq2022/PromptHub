# Proposal

## Why

桌面端 `项目 Skills` 当前存在两类体验问题：

- 页面三栏的头部节奏和分割线没有对齐，视觉上显得错位。
- 添加项目流程过于手动：要先填名称，再选目录，再手动扫描；项目卡片也缺少更清晰的项目标识。

## Scope

- 修复 `项目 Skills` 视图的头部与分栏对齐问题。
- 优化“添加项目”流程：先选项目根目录、自动填充项目名称、添加后自动扫描。
- 为项目列表和项目头部增加更明确的首字母标识。

## Risks

- 会改动桌面端 Skills 项目视图的布局和交互顺序。
- 会影响 `skillProjects.scanPaths` 在设置 store 中的展示语义。

## Rollback

- 恢复 `SkillProjectsView.tsx` 的原始弹窗字段顺序与项目列表样式。
- 恢复创建项目后不自动触发扫描的行为。
