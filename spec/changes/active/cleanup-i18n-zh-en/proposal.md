# Proposal: 国际化语言包清理（仅保留中英）

## Why

随着项目的迭代，多语言（7 种语言）的维护成本逐渐升高，且绝大部分用户仅使用中文和英文。维护德语、西班牙语、法语、日语和繁体中文需要大量的翻译同步工作，给日常功能的开发（测试对齐断言）带来了额外负担。通过清理不常用的 5 种语言，可以简化多语言管理流程，降低开发心智负担，并加快构建和运行测试的速度。

## Scope

- **In scope**:
  - 物理删除桌面端及 Web 端非中英的翻译 JSON 文件 (`de`, `es`, `fr`, `ja`, `zh-TW`)。
  - 修改 `apps/desktop` 和 `apps/web` 下的 i18n 配置和语言检测函数，仅保留 `en` 与 `zh`。
  - 修改 `settings.store.ts` 与相关的 UI 设置组件，将可选择的语言缩减为“简体中文”与“English”。
  - 更新单元测试、集成测试以及 smoke 测试，剔除对已被删除语言包的导入和对齐断言。
  - 调整 `AGENTS.md` 中的多语言同步规程。

- **Out of scope**:
  - 暂不处理文档中（如 `website/src/content/docs`）的多语言文件，只关注核心应用和测试的运行态代码与配置。

## Risks

- **系统检测语言回退**：若用户的系统语言为日语或德语，清理后应能正常且安全地回退到英文（`en`），不能引起初始化崩溃。
- **单元测试破损**：测试脚本中有大量的静态 `import`，删除多语言文件会导致 webpack/vite 编译失败。必须同步清理所有测试导入。

## Rollback Thinking

- 在 Git 中可以非常容易地通过回滚提交来还原已删除的 JSON 文件与配置。
