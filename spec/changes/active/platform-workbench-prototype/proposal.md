# Proposal

## Why

PromptHub 当前主心智仍然是 `Prompt + Skill` 双对象管理。

为了评估后续扩展到 `Agent`、`MCP`、`Workflow`、`Resource` 等对象时的信息架构与视觉密度，需要先在不破坏现有主界面的前提下提供一个可直接预览的桌面端 UI 原型页。

## Goals

- 提供一个可在应用内直接访问的“平台工作台”高保真原型页。
- 展示未来多资源管理平台的导航分层、列表区、详情区、全局搜索与状态卡片布局。
- 不替换现有主界面，只作为预览入口，用于评估方向。

## Non-Goals

- 本轮不改动 Prompt / Skill 的主路由与现有业务逻辑。
- 本轮不实现真实的 Agent / MCP 数据模型。
- 本轮不引入数据库迁移或新的持久化状态。

## User Flow

1. 用户打开设置页。
2. 用户点击 `Platform Preview` / `平台预览`。
3. 用户看到一套新的多资源平台式工作台原型，包括：
   - 左侧分组导航
   - 顶部全局搜索与状态摘要
   - 中部资源列表
   - 右侧上下文详情面板
4. 用户通过点击左侧导航项与资源卡片，预览未来产品的结构方向。
