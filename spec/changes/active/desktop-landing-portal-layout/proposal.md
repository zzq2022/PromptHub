# 桌面端落地门户与组合导航重构提案

## 1. 背景与目标
当前 PromptHub 的桌面端布局采用了“窄侧边栏 (Rail) + 宽侧边栏 (Panel) + 主内容区”的传统两层式导航结构。用户希望对桌面端的整体页面布局进行重构，并参考 Kimi 的高品质界面设计：
- **统一轻量侧边栏**：左侧统一为单个侧边栏，顶部放置 `Work`（工作区：提示词/技能/系统规则）与 `Chat`（对话：智能体项目及会话）双模式切页标签，取代现有散落的 Rail 项。
- **精美落地门户 (Portal Dashboard)**：当没有任何 Prompt、Skill 或会话被选中时，在主内容区渲染一个类似 Kimi 的精美落地门户——上方居中展示 Logo，中间提供一个带微动画的浮动搜索/聊天框，下方提供分类 Tab（如通用、文本、绘图、项目、系统 rules 等）和卡片网格。
- **顺滑的工作区切入**：点击卡片或在输入框发起交互时，无缝过渡到具体功能的工作分栏界面。

## 2. 核心架构与模块边界
- **应用模式分栏**：
  - `Work` 模式：承载 Prompt 库、Skill 库和 Rules 管理。左侧展示对应的目录树和过滤项。
  - `Chat` 模式：承载 Agent 实时对话。左侧展示 Agent 项目列表和当前项目的 `历史会话`，支持快捷新建会话。
- **落地门户驱动**：
  - 新建 `PortalDashboard.tsx` 承担中央控制台职责。通过读取 PromptStore、SkillStore 和 RulesStore 获取最近编辑/常用/推荐条目，进行跨模块的卡片式渲染。

## 3. 风险与回滚方案
- **风险**：多模块联动（Prompt/Skill/Rules）在同一侧边栏渲染时可能导致状态管理复杂、Zustand 订阅冗余引起卡顿。
- **回滚方案**：新布局采用开关控制或保留原侧边栏组件的向后兼容，一旦发生重大阻塞性 UI 崩溃，可通过设置一键切换回经典双栏布局。
