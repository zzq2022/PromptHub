# Proposal: 系统重构——实现 Codex 页面、双向自动同步、SkillHub 商店与多租户权限隔离

## Why

随着 PromptHub 功能的扩展，用户对“本地 AI 协同开发”以及“团队提示词/技能共享”的需求日益增长。为提高本产品的工程价值与用户体验，我们需要进行一次全栈级的重大功能重构：
1. **桌面端 UI 重构**：现有的列表-详情布局较为传统，需借鉴 Codex 的高度专业化开发界面（三栏式布局、工程目录树、多文件交互与命令行辅助），建立更加贴合开发者直觉的工作区。
2. **全生命周期的技能管理**：目前本地技能与外部商店的交互偏向单向导入，需将其完整嵌入桌面端核心功能中（实现查询、收藏、删除、安全扫描，并提供图形化发布通路）。
3. **高可用双向自动同步**：从手动的导入/导出升级为实时的双向静默增量同步，降低多机协作成本。
4. **模型与工具组合管理**：支持多模型统一配置，并在输入窗口下方允许用户按需组装/切换本地 Skill 和外部 MCP 工具。
5. **SkillHub 商店（类 ClawHub 体验）**：在服务端提供公开的可检索技能目录、评分和安全等级展示，为 PromptHub 建立真正的应用生态。
6. **多租户与管理员审核工作流**：在自托管 Web 服务中彻底隔离普通用户与管理员，提供完整的发布申请-审核机制。

## Scope

- **In scope**:
  - **桌面端 (apps/desktop)**:
    - 重构主界面为 **Codex 风格编辑器**。包含三栏布局（侧边栏工程目录/规则/分类、主编辑器、大模型交互区域）。
    - 嵌入完整的技能管理面板（查询、收藏、本地删除、AI 安全合规扫描）。
    - 整合 SkillHub（ClawHub 克隆版）客户端浏览和一键下载/安装页面。
    - 实现**后台周期性自动同步服务**（双向 Push/Pull，防抖触发，同步状态指示灯）。
    - 开发**模型管理面板 (Model Provider Config)**，支持 OpenAI/Anthropic/DeepSeek 等提供商管理。
    - 升级输入窗口底部的状态配置栏：除了项目选择，新增“技能与 MCP 选择器”（以 Badge/Tag 形式展示已加载工具，并动态配置底层 LLM 会话的 Tools/System Prompt）。
  - **服务端 (apps/web)**:
    - 依照 ClawHub 规范，开发公共技能商店（SkillHub）页面：支持搜索、分类筛选、周下载量、标星数以及 AI 安全扫描评级显示。
    - 服务端用户分权系统：
      - **普通用户**：提供隔离的设置中心、私有提示词/技能同步、提交技能发布申请的 `PUT /skills/:id/publish` 接口。
      - **管理员 (Admin)**：提供审核中心 Dashboard。可分页列出所有 `pending` 状态的技能发布申请，执行批准（将其 `visibility` 自动变更为 `shared` 并公开）或拒绝的操作。
  - **核心包 (packages/core & packages/db)**:
    - 扩展 `skills` 表模型，完善审核相关字段；新增自动同步的定时任务/防抖同步引擎。

- **Out of scope**:
  - 本次重构不涉及 CLI 工具（`apps/cli`）交互界面的重构（但 CLI 底层依赖的核心包 API 会保持兼容）。

## Risks

- **自动同步冲突风险**：多端同时修改相同的数据行可能导致覆盖冲突。设计中需强依赖 `updatedAt` 时间戳进行 LWW (Last-Write-Wins) 处理，并在冲突严重时回退到手动合并。
- **并发性能风险**：桌面端引入自动同步后，在频繁编辑提示词时，若防抖机制不当可能导致高频的服务器请求和网络风暴，影响系统流畅度。

## Rollback Thinking

- 在重构前，将当前主干分支备份为 `release-v0.5.7-legacy` 分支。
- 数据库结构升级保留 `init.ts` 事务备份，旧库升级失败时可直接通过时间戳备份快速还原。
