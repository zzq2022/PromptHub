# Proposal

## Why

桌面端当前还有四处体验没有闭环：

- 数据设置里的升级回滚快照列表会随着备份数量增长直接拉长页面，没有默认收敛，也没有独立滚动区域。
- 外观设置只能调主题和字号，无法使用本地图像作为应用背景，也无法控制透明度与虚化强度。
- 图片模式已经具备基础玻璃态，但 panel / search / sidebar / card 仍停留在“半透明面板”层级，缺少更细的 Liquid Glass 材质合同与清晰的海拔梯度。
- Skill 详情页导出仍暴露 `JSON` 这种偏内部交换格式，和用户真正关心的 `SKILL.md` / 整个仓库文件夹导出不一致。

这几项都属于“桌面端本地资产管理与沉浸式外观”不完整：恢复快照缺少合适浏览容器，背景图缺少稳定设置模型，图片模式缺少成体系的液态玻璃语义层，Skill 仓库缺少完整导出能力。

## Scope

- In scope:
- 为升级快照列表增加默认收敛、展开/收起、独立滚动区域与摘要文案
- 为桌面端外观设置增加背景图选择、清除、透明度与虚化控制
- 为图片模式补一轮独立的 Liquid Glass 质感优化，拆分 blur / tint / stroke / specular / shadow 合同
- 为 `TopBar` 搜索框、`Sidebar`、Prompt 列表 pane / card 与相关 toolbar 接入局部材质层和更明确的层级梯度
- 复用现有图片存储链路保存背景图文件，避免把大图直接写入 localStorage
- 将 Skill 导出收敛为 `SKILL.md` 与 `zip` 两种桌面端导出能力
- 保持桌面端 Skill 安装/扫描配置为独立设置页签，不并入 `DataSettings`
- Skill 平台顺序支持直接拖拽调整，并将默认展示顺序收敛为 Claude / Codex / OpenCode / OpenClaw / Hermes / Cursor 优先
- GitHub issue 提交时自动根据表单中的版本字段补一个 `version: ...` 标签，避免手动维护版本标签
- 补充相应 IPC、前端工具函数、测试与 i18n 文案

- Out of scope:
- Web 运行时的背景图上传能力
- 壁纸驱动的动态折射、噪点、实时 parallax 等高成本动效
- Skill JSON 导入/导出协议重构
- 数据恢复主流程本身的恢复语义调整

## Risks

- 若背景图直接写入 localStorage，会带来配额、启动性能和备份体积问题。
- 若 Liquid Glass 优化只停留在单个组件覆写，而没有独立 token 合同，后续主题市场或用户自定义会继续和默认 UI 耦合。
- 若局部材质层级过强，会在亮色壁纸下压低正文对比度，造成“好看但难读”。
- 若 Skill zip 导出只在前端拼装文本，可能漏掉二进制或附加文件，导致“整个文件夹导出”名不副实。
- 若升级快照列表交互改动过大，可能影响现有恢复入口的可发现性。

## Rollback Thinking

- 背景图能力基于新增设置字段与可选文件名；即使回退代码，旧设置也只会被忽略，不会破坏其他设置。
- Liquid Glass 质感优化只重构 renderer 样式合同与组件 class 组合，不改变背景图持久化模型；必要时可以单独回退视觉层而保留背景图功能。
- Skill zip 导出通过新增 IPC 通道实现，不修改现有 `skill:export` 文本导出语义；出现问题时可单独回退 zip 能力。
- 回滚列表的收敛仅影响展示方式，不改变实际恢复/删除动作。
