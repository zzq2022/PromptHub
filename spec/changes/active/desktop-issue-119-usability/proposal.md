# Proposal

## Why

GitHub issue `#119` 提了三个桌面端易用性需求：

- 文件夹栏与提示词列表栏支持拖拽调整宽度
- 卡片模式下双击用户提示词即可进入编辑
- 画廊 `S/M` 模式下超长标题应自动换行，避免两行截断影响阅读

当前代码里，第一个需求已经落地并有独立列宽测试，但后两个需求仍未闭环：

- 卡片详情内联编辑目前只能通过双击标题进入，用户提示词区域还不能直接作为入口
- 画廊卡片标题仍使用两行 `line-clamp`，在 `S/M` 模式下会过早截断

## Scope

- In scope:
- 允许桌面卡片详情面板中的用户提示词区域双击进入现有 inline edit 流程。
- 调整桌面画廊 `small` / `medium` 模式下的标题展示，让超长标题自动换行而不是两行截断。
- 为上述行为补回归测试。

- Out of scope:
- 重新设计完整 prompt 编辑器。
- 调整数据库、IPC 或同步契约。
- 修改已存在的列宽拖拽能力本身，除非为了验证 issue 范围需要补文档说明。

## Risks

- 放开标题换行会改变画廊卡片高度节奏，可能带来更松散的网格视觉。
- 用户提示词区域双击进入编辑需要避免与现有 Markdown/plain-text toggle、复制等操作冲突。

## Rollback Thinking

- 如果画廊标题完全放开换行导致视觉噪声过高，可以回退到更宽松但有限的多行截断策略。
- 如果用户提示词双击入口与现有交互冲突，可以只保留标题双击入口。
