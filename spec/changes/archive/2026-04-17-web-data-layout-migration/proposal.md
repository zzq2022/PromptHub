# Proposal

Archived from historical planning content in `spec/changes/legacy/docs-08-todo/web-data-layout-migration.md`.

## Summary

Migrate the web app to a new data layout aligned with the desktop-era `data/ + config/ + backups/ + logs/` model and newer prompt file formats.

## Why

- Reduce structural drift between desktop and web
- Improve recoverability and future sync behavior
- Remove legacy layout complexity from the web runtime
